# Copy and Localization Spec

## 1. 목적
이 문서는 모바일 앱의 문구 스타일과 다국어 규칙을 정의한다.
한국어를 기본 언어로 두되, 영어 전환 시 동일한 정보 구조와 의미를 유지해야 한다.

## 2. 언어 정책
- 기본 언어: 한국어
- 보조 언어: 영어
- 버튼 라벨은 짧게 유지
- 서비스 이름은 번역하지 않음
  - `Spotify`
  - `YouTube Music`
  - `YouTube MV`

## 3. 제목 규칙
- 화면 제목은 기능 중심
- 내부 구현 용어 금지
- 예:
  - `이번 달 컴백 캘린더`
  - `레이더`
  - `검색`
  - `팀 페이지`

금지 예:
- `주간 컴백 인텔리전스 사이클`
- `검토 필요`
- `추적 대상`

## 4. 버튼 라벨 규칙
### 4.1 Primary
- `팀 페이지`
- `상세 보기`
- `앨범 보기`

### 4.2 Service
- 표면 라벨은 서비스 이름만 사용
- `검색 열기`는 tooltip 또는 보조 문구에만 사용

### 4.3 Meta
- `기사 원문`
- `공식 공지`
- `소스 보기`

## 5. Empty State Copy
- 빈 날짜: `이 날짜에는 등록된 일정이 없습니다.`
- 예정 컴백 없음: `등록된 예정 컴백이 없습니다.`
- 최근 앨범 없음: `표시할 최근 앨범이 없습니다.`
- 검색 결과 없음: `검색 결과가 없습니다.`
- 트랙 정보 없음: `트랙 정보가 아직 준비되지 않았습니다.`

## 6. Error Copy
- 공통: `정보를 불러오지 못했습니다.`
- retry CTA: `다시 시도`
- 부분 실패: `일부 정보만 표시됩니다.`

## 7. Date Copy
- 한국어: `2026년 4월 13일`
- 목록 축약: `4월 13일`
- 카운트다운: `D-3`, `오늘`, `내일`
- 모호한 날짜는 `4월 예정`, `날짜 미정` 규칙 사용

## 8. Status Chip Copy
- `예정`
- `확정`
- `루머`
- `싱글`
- `미니`
- `정규`
- `EP`
- `OST`
- `콜라보`
- `타이틀`

## 9. Search Copy
- placeholder: `팀, 앨범, 곡, 별칭 검색`
- 탭: `팀`, `발매`, `예정`
- 최근 검색 섹션: `최근 검색`

## 10. Localization Rules
- 서비스명, 팀명, 고유 릴리즈명은 원문 유지 가능
- 한국어 환경에서도 일본어/영어 곡명은 원문 유지 우선
- 날짜/상태/버튼 문구는 언어에 맞춰 번역
